# 0005 — Codify the real layout; kill the fiction

**Status:** Accepted · 2026-07-02

## Context

The structure docs described a codebase that doesn't exist. `ARCHITECTURE.md`
(itself uncommitted) documented a `src/domains/` + `src/platform/` + `src/shared/`
layout and a `pnpm lint:lines` script — **none of which exist**. The real layout
is `src/{api,auth,config,mcp,tools,schemas,skills,scripts,types,utils}` plus
`index.ts` / `server-http.ts`. The stale `biome.json` override enforced size caps
on those same non-existent `domains|platform|shared` directories, so it enforced
nothing (see [0001](0001-formatter-biome.md)).

`tsconfig.json` also excluded a hand-written source file
(`src/api/marketing-and-promotions/marketing.ts`) from typechecking.

## Decision

- **Codify the real layout.** Rewrite `ARCHITECTURE.md` to the actual tree and
  pipeline; add `CONTEXT.md` (orientation) and `LANGUAGE.md` (glossary). Delete
  the `domains/platform/shared` fiction and the dead Biome override. **No code
  moves** — the docs are made to match the code, not vice-versa.
- **No hand-written source excluded from typecheck.** Un-exclude `marketing.ts`;
  verified it (and the generated types it pulls in) typechecks cleanly, so the
  gate stays green.

## Consequences

- Two files remain intentionally excluded from `tsconfig.json`, for reasons that
  are **not** "hide broken hand-written code":
  - `src/types/**` generated response/type files — regenerated by `pnpm run sync`;
    excluding them keeps typecheck robust against generated output.
  - `src/scripts/test-endpoints.ts` — a legacy dev script with ~29 pre-existing
    type errors; out of scope for this change set (tracked follow-up).
- Discovered during this work and recorded for honesty: `biome.json`,
  `ui/tsconfig.json`, and `ARCHITECTURE.md` were all **untracked** local files —
  a fresh clone had none of them. `biome.json` and `ARCHITECTURE.md` are now
  committed; `ui/tsconfig.json` is tracked as a follow-up in
  [0004](0004-ci-workflow-architecture.md).
- Style intent is in [CODE-STYLE.md](../../../CODE-STYLE.md); size-cap
  exemptions match the declarative dirs listed there and in
  [0001](0001-formatter-biome.md).
